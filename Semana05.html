Bases de datos Northwind

Script 1

-- 1. Crear la base de datos para los ejercicios
CREATE DATABASE UTP_SD
USE UTP_SD

-- 2. Crear la tabla de auditoría
CREATE TABLE Auditoria (
    IdAuditoria     INT IDENTITY(1,1) PRIMARY KEY,
    FechaAuditoria  DATETIME DEFAULT GETDATE(),
    TipoRegistro    CHAR(1),
    CustomerID      CHAR(5),
    CompanyName     VARCHAR(100),
    ContactName     VARCHAR(100),
    ContactTitle    VARCHAR(100)
)

-- 3. Modificar el tamaño de un campo (aunque ya se define como CHAR(5) al crear la tabla, se repite)
ALTER TABLE Auditoria
ALTER COLUMN CustomerID CHAR(5)

-- 4. Listar datos de otra base de datos (Northwind)
SELECT *
FROM Northwind.dbo.Customers

-- 5. Insertar un registro en la tabla de auditoría
INSERT INTO Auditoria (TipoRegistro, CustomerID, CompanyName, ContactName, ContactTitle)
VALUES ('I', 'UTP01', 'UTP', 'Docente', 'Docente SD')

-- 6. Insertar otro registro desde otra base y listar los datos
INSERT INTO UTP_SD.dbo.Auditoria (TipoRegistro, CustomerID, CompanyName, ContactName, ContactTitle)
VALUES ('I', 'UTP02', 'UTP SD', 'Docente del curso', 'Docente SD')

SELECT * FROM UTP_SD.dbo.Auditoria

-- 7. Insertar un cliente en la tabla Customers
INSERT INTO Customers (CustomerID, CompanyName, ContactName, ContactTitle)
VALUES ('CL003', 'Cliente 003', 'Maria Zapata', 'Jefe de Compras')

-- 8. Crear un trigger para insertar en auditoría cada vez que se inserte un nuevo cliente
CREATE OR ALTER TRIGGER trg_Customers_Nuevo
ON Customers
FOR INSERT
AS
BEGIN
    INSERT INTO UTP_SD.dbo.Auditoria (TipoRegistro, CustomerID, CompanyName, ContactName, ContactTitle)
    SELECT 'I', CustomerID, CompanyName, ContactName, ContactTitle
    FROM Inserted
END

Este script realiza lo siguiente:

  1. Crea una base de datos llamada UTP_SD
    • Para trabajar con un entorno de pruebas y no afectar datos reales.
  1. Crea una tabla llamada Auditoria
    • Esta tabla almacenará los cambios o inserciones que ocurran sobre la tabla Customers.
    • Campos:
      • IdAuditoria: clave primaria autoincremental.
      • FechaAuditoria: se llena automáticamente con la fecha actual.
      • TipoRegistro: tipo de operación ('I' para insert).
      • CustomerID, CompanyName, ContactName, ContactTitle: datos básicos del cliente registrados.
  1. Modifica (innecesariamente en este caso) el tamaño del campo CustomerID
    • Ya tenía el tamaño correcto, se repite el cambio como seguridad.
  1. Consulta los clientes de otra base de datos (Northwind)
    • Sirve para verificar o copiar información de referencia.
  1. Inserta registros en la tabla Auditoria manualmente
    • Simula acciones como si se estuvieran insertando desde Customers.
  1. Inserta registros en Customers
    • Por ejemplo: CL003 es un nuevo cliente.
  1. Crea un trigger (trg_Customers_Nuevo)
    • Cada vez que alguien inserta un nuevo cliente en Customers, el trigger guarda automáticamente una copia del registro en la tabla Auditoria con tipo 'I' (inserción).

Script 2

-- 1. Crear trigger que audita inserciones en Customers
CREATE OR ALTER TRIGGER trg_Cliente_Nuevo
ON Customers
FOR INSERT
AS
BEGIN
    INSERT INTO UTP_SD.dbo.Auditoria (TipoRegistro, CustomerID, CompanyName, ContactName, ContactTitle)
    SELECT 'I', CustomerID, CompanyName, ContactName, ContactTitle
    FROM Inserted
END

-- 2. Transacción de prueba para insertar cliente (sin commit para prueba)
BEGIN TRAN
INSERT INTO Customers (CustomerID, CompanyName, ContactName, ContactTitle)
VALUES ('00003', 'Cliente 03', 'Nombre Cliente 03', 'Cargo Cliente 03')

SELECT * FROM Customers
-- ROLLBACK o COMMIT según prueba
ROLLBACK

-- 3. Crear trigger para auditoría de actualizaciones en Customers
CREATE OR ALTER TRIGGER trg_Cliente_Actualiza
ON Customers
FOR UPDATE
AS
BEGIN
    -- Se registra la versión anterior del cliente actualizado
    INSERT INTO UTP_SD.dbo.Auditoria (TipoRegistro, CustomerID, CompanyName, ContactName, ContactTitle)
    SELECT 'U', CustomerID, CompanyName, ContactName, ContactTitle
    FROM Deleted
END

-- 4. Probar actualización dentro de una transacción
BEGIN TRAN
UPDATE Customers
SET ContactTitle = 'Gerente'
WHERE CustomerID = '00003'
-- ROLLBACK o COMMIT según prueba
ROLLBACK

Este script se enfoca más en:

  1. Trigger trg_Cliente_Nuevo
    • Al igual que el anterior, audita toda nueva inserción en Customers y la guarda en Auditoria.
  1. Uso de transacciones (BEGIN TRAN)
    • Se inserta un cliente y se prueba con ROLLBACK para anular los cambios. Esto permite ver cómo se comporta el trigger sin modificar los datos definitivamente.
    • Sirve para pruebas sin afectar la base de datos permanentemente.
  1. Creación de trigger trg_Cliente_Actualiza
    • Este trigger registra cualquier actualización hecha a la tabla Customers.
    • Guarda en la tabla de auditoría los valores antiguos (gracias a la tabla Deleted), y marca el tipo como 'U' (update).
    • No guarda los nuevos valores, solo lo que había antes del cambio.
  1. Otra prueba de transacción con UPDATE
    • Se prueba actualizar un campo (ContactTitle) y luego se hace ROLLBACK.
    • Esto permite verificar si el trigger captura correctamente los datos antes de la actualización.

Extra – Limpio y separado por bloques

1. Consulta para mostrar diferencias entre registros antiguos y nuevos

SELECT
    NR.IdAuditoria,
    NR.FechaAuditoria,
    NR.CustomerID,
    NR.CompanyName,
    NR.ContactName,
    NR.ContactTitle,
    RM.CustomerID AS CustomerID_Anterior,
    RM.CompanyName AS CompanyName_Anterior,
    RM.ContactName AS ContactName_Anterior,
    RM.ContactTitle AS ContactTitle_Anterior,
    -- Mostrar diferencia en el cargo
    IIF(NR.ContactTitle = RM.ContactTitle, '', RM.ContactTitle + ' --> ' + NR.ContactTitle) AS Cargo_Modificado
FROM Auditoria AS NR WITH (NOLOCK)
INNER JOIN (
    SELECT *
    FROM Auditoria AS RM WITH (NOLOCK)
) AS RM
    ON NR.ReferenciaIdAuditoria = RM.IdAuditoria
WHERE NR.ReferenciaIdAuditoria IS NOT NULL

Esta parte realiza una comparación entre registros:

  • Consulta avanzada para mostrar cambios entre versiones de un mismo cliente.
    • Se hace un JOIN entre la tabla Auditoria y ella misma, comparando el registro nuevo (NR) y el anterior (RM).
    • Se verifica si el ContactTitle ha cambiado.
    • Si cambió, muestra una cadena como: "Gerente --> Jefe de Compras".
  • Filtro con ReferenciaIdAuditoria
    • Parece que la tabla Auditoria está pensada para tener un campo ReferenciaIdAuditoria que indica cuál fue la versión anterior (aunque ese campo no está en el script de creación).
    • Esto permite rastrear el historial de cambios por cliente o acción.
